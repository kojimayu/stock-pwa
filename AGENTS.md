# STOCK-PWA 開発監視体制 (2026)

## プロジェクト・コンテキスト
- **運営**: 株式会社プラスカンパニー
- **用途**: エアコン設置現場での在庫管理（PWA）
- **環境**: 屋外、スマートフォン操作、オフライン環境あり
- **目的**: 材料費高騰に伴う正確な在庫管理と、現場の事務負担軽減

## エージェント定義
1. **Architect (Main)**: 全体の実装。
2. **Security (Claude 4.5 Sonnet Thinking)**: 
   - 外部視点での脆弱性診断。Azure接続情報の保護。
3. **QA/Logic (Gemini 3 Pro High)**: 
   - 在庫管理ロジックの整合性確認。Azure同期処理のデグレード監視。
4. **UX/Design (Gemini 3 Flash)**: 
   - 役割: ユーザーインターフェースの品質管理、色彩設計、レスポンシブ性の検証。
   - デバイス想定:
     - **管理画面 (基本)**: PC操作 (詳細な表、標準入力)。
     - **棚卸・発注業務**: スマホ操作 (Admin内だが現場で使用。片手操作、大きなタッチターゲット)。
     - **Kiosk (倉庫・現場)**: タブレット入力 (視認性重視、スキャナー併用)。
   - 監視項目: 現場での操作性、オフライン表示の適切さ。

## 運用ルール
- 変更を加える際は、まずQAエージェントが「既存ロジックへの影響」を分析すること。
- セキュリティエージェントが警告を出した場合は、実装を即座に中断し対策を協議する。
- デザインエージェントは、屋外視認性と片手操作性を最優先して評価する。
- 全てのレビュー結果を Artifacts（成果物）として出力し、人間の承認後に反映する。
